容器环境配置安装

mpi4py

mpi4py 是 Python 的一个 MPI（Message Passing Interface，消息传递接口）库，主要用于高性能计算领域，可实现并行计算功能。

为什么要强制从源代码编译？

–no-binary：该选项会让 pip 放弃使用预编译的二进制包（也就是 wheel 文件），转而强制从源代码开始编译安装包。

mpi4py 是一个对底层 C 库有依赖的 Python 扩展模块，它依赖于 MPI 实现（像 OpenMPI、MPICH 这类）。

预编译二进制包的局限性：预编译的 mpi4py wheel 文件一般是和特定的 MPI 实现以及版本绑定在一起的。要是系统里的 MPI 环境和预编译包所依赖的环境不兼容，就会出现运行时错误

通过从源代码编译，mpi4py 能够和系统中已有的 MPI 环境（像系统自带的 OpenMPI）进行动态链接，这样就能保证兼容性。

要成功从源代码编译 mpi4py，系统中必须提前安装好以下工具和库：sudo apt-get install libopenmpi-dev openmpi-bin && sudo apt-get install build-essential python3.10-dev

如何验证是否安装成功

python -m pip list | grep mpi4py

或者

python -c "import mpi4py; print(mpi4py.__version__)"
