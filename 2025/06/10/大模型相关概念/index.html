TensorRT-LLM

NVIDIA推出的开源库

TensorRT-LLM是NVIDIA推出的一个开源库，用于定义、优化和执行大型语言模型 (LLM) 在生产环境的推理。

TensorRT-LLM是NVIDIA用于做 LLM 的可扩展推理方案。该方案基于 TensorRT 深度学习编译框架来构建、编译并执行计算图，并借鉴了FastTransformer 中高效的 Kernels 实现，然后利用 NCCL 完成设备之间的通讯。

TensorRT-LLM开源程序库可加快 NVIDIA GPU 上大语言模型的推理性能。

关于TensorRT-LLM的使用，首先获得一个预训练好的模型，然后利用 TensorRT-LLM 提供的 API 对模型计算图进行改写和重建，接着用 TensorRT 进行编译优化，然后保存为序列化的 engine 进行推理部署。

例如Llama，首先安装TensorRT-LLM，下载预训练模型，接着利用 TensorRT-LLM 对模型进行编译，最后进行推理。

对于模型推理的调试，TensorRT-LLM 的调试方式与 TensorRT 一致。由于深度学习编译器，即 TensorRT，提供的优化之一是 layer 融合。因此，如果要输出某层的结果，就需要将对应层标记为输出层，以防止被编译器优化掉，然后与 baseline 进行对比分析。同时，每标记一个新的输出层，都要重新编译 TensorRT 的 engine。

对于自定义的层，TensorRT-LLM 提供了许多 Pytorch-like 算子帮助用户实现功能而不必自己编写 kernel。如样例所示，利用 TensorRT-LLM 提供的 API 实现了 rms norm 的逻辑，TensorRT 会自动生成 GPU 上对应的执行代码。

特点：

性能极致：基于 C++ 开发，有 NV 算子开发黑魔法加持，在其重点支持的场景里，性能可以做到极致。

优化技术丰富：提供自定义 Attention Kernel、Inflight Batching、Paged KV Caching、量化技术（FP8、INT4 AWQ、INT8 SmoothQuant 等）等多种优化技术，充分发挥 NVIDIA GPU 的推理性能。

部分开源：相比 vLLM 和 SGLang，其开源程度有限，这可能会对一些需要深度定制和了解内部原理的用户造成一定限制。

vLLM

vLLM 是一个基于 Python 开发的开源 LLM 推理引擎

特点：

技术创新：Paged Attention 技术创新性地解决了高吞吐场景下的性能问题。

兼顾性能和易用性：PA kernel 用 CUDA 开发，其余部分如 Batch 调度、模型定义、并行推理等使用 PyTorch 开发，在一定程度上兼顾了性能和灵活性，降低了开发者参与门槛。

开源社区成熟：开源社区经营方式成熟，汇聚了模型开发者、使用者和技术创新者等。

多硬件支持：支持 AMD、Intel 等厂商的 GPU，具有更广泛的硬件适应性。

了解 Paged Attention 技术的原理和优势，掌握 vLLM 的 Python API 使用方法，包括如何进行模型加载、推理请求的发送以及结果的处理等。同时，可以深入研究其基于 PyTorch 的架构设计，以便于进行二次开发和优化。

SGLang

SGLang 是 LMSYS Org 团队开发的一个用于 大型语言模型和视觉语言模型的快速服务框架，于 2025 年 1 月正式推出，采用 Apache 2.0 许可授权。

特点：

性能卓越：在运行 Llama 3.1 405B 等模型时，其吞吐量和延迟表现优于 vLLM 和 TensorRT-LLM，在某些情况下，吞吐量能达到 TensorRT-LLM 的 2.1 倍，vLLM 的 3.8 倍。

用户友好、易于定制：轻量级和模块化内核，完全用 Python 编写，核心调度器不到 4K 行代码，易于用户进行定制和修改。

集成多种技术：基于并增强了多个开源 LLM 服务引擎的优秀设计，利用了 FlashInfer 的高性能注意力 CUDA 内核，集成了受 gpt - fast 启发的 torch.compile，还引入了 RadixAttention 和压缩状态机等创新技术。

学习 SGLang 的 DSL（领域特定语言），了解如何通过前端定义来控制 LLM 的生成过程。深入研究其后端 Runtime 的核心创新优化点，如 RadixAttention 技术对于 KV 缓存重用的原理和实现方式。掌握 SGLang 与其他开源引擎的集成方法，以及如何利用其进行高效的模型服务。
